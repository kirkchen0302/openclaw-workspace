## audience_dashboard 定義修正（重要）

### 919 載具（載具驗證失敗）

**定義**：財政部回應代碼 919，代表該會員的手機載具驗證碼錯誤，無法同步發票資料。

**正確判斷方式**：
```sql
-- 正確（最終版）
JOIN (
  SELECT carrier_hk, is_active, mof_status,
    ROW_NUMBER() OVER (PARTITION BY carrier_hk ORDER BY effective_from DESC) AS rn
  FROM `production-379804.intermediate.sat__carrier`
) c ON lmc.carrier_hk = c.carrier_hk
WHERE c.rn = 1
  AND c.is_active = TRUE
  AND c.mof_status = 'INACCURATE_AVAILABILITY'
```

**欄位說明**：
- `is_active`：載具條碼是否存在（FALSE = 已被刪除，不納入）
- `mof_status`：財政部驗證結果，sat__carrier 有多筆歷史，需取最新一筆
- `INACCURATE_AVAILABILITY`：財政部回傳 919，驗證碼錯誤

**舊（錯誤）方式們**：
```sql
-- ❌ v1：barcode 格式判斷，完全錯誤
WHERE hc.barcode LIKE '/%'

-- ❌ v2：沒取最新一筆，且 is_active 邏輯搞反
WHERE c.mof_status = 'INACCURATE_AVAILABILITY'

-- ❌ v3：is_active=FALSE 是「已刪除載具」，不是 919
WHERE c.is_active = FALSE
```

### 活躍使用者定義

**定義**：近 30 天內有 `session_start` event 的會員視為「活躍」。

- 依據：`base__sat__session_session_start_activity` table
- 不參考其他 event（如 page_view、purchase 等）
- 快照基準日：`2026-03-22`（固定時間點，不隨日期變動）
