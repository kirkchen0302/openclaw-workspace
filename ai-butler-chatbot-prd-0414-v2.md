## 1. 版本差異摘要（vs 0414_v1）

| 面向 | 0414_v1 | 0414_v2 |
|------|---------|---------|
| Hook 數量 | 6 個 | **3 個固定**（精簡聚焦） |
| Hook 層級 | 3 層（Hook→追問→深追問） | **2 層**（Hook→追問，無深追問） |
| 訂閱 vs 重複消費 | 混合呈現 | **分開**：訂閱=你知道的，重複=你不知道的 |
| 受眾分群 | 通路佔比 3 種 | **品項+通路 12 種**受眾標籤 |
| 速食/壽司店 | 部分按品項計 | **全部按發票（visit）計** |
| 重複門檻 | ≥3 次 | **≥5 次**（真正的習慣） |
| 省錢建議 | 通用 | **品類專屬**（速食/咖啡/零食/乳製品/美妝各有策略） |
| 換算比較 | 可能用同品類 | **排除正在省的品類** |
| 訂閱偵測 | 基本關鍵字 | **通路×品項配對 + regex** |

## 2. 架構：3 個固定 Hook

### 2.1 開場

```
「嗨，我已經看完你的 {N} 張發票和所有品項明細了。選一個你好奇的問題，我來幫你分析 👇」
```

一句話 + 直接顯示 3 個 Hook 按鈕。

### 2.2 Hook 總覽

| # | Hook 按鈕 | ID | 核心問題 |
|---|---------|-----|---------|
| 1 | 「我的訂閱和重複消費花了多少？」 | `recurring` | 自動化消費的全貌 |
| 2 | 「我的消費有什麼隱藏規律？」 | `pattern` | 你不知道的行為模式 |
| 3 | 「我的『{品類}』花了多少？能省嗎？」 | `topcategory` | 最大品類 + 省法 + 跟人比 |

---

## 3. Hook 1：訂閱 + 重複消費

### 3.1 Insight 卡片

**大數字**：年度合計（訂閱 + 重複消費）
**排行**：訂閱（📱標記）在前，重複品項在後，明確區分

### 3.2 訂閱偵測邏輯

**Method 1 — 品項關鍵字**：
```
月費, 年費, 月訂閱, 季訂閱, 年訂閱, 訂閱費, 訂閱制,
訂閱方案, 訂閱服務, 訂閱月費, 訂閱年費
```

**Method 1b — Regex**：
```
/(youtube|spotify).*premium/i
/^youtube$/i
```

**Method 2 — 通路 × 品項配對**：

| 通路 | 品項關鍵字 |
|------|----------|
| ubereats / 優食台灣 | uber one, 訂閱 |
| foodpanda / 富胖達 | pandapro, 訂閱 |
| 酷澎 / coupang | wow, 會員, 訂閱 |
| spotify | premium, 訂閱, 月費 |
| netflix | 訂閱, 月費, netflix |
| disney / 迪士尼 | 訂閱, 月費 |
| apple / itunes | icloud, 訂閱, 月費 |
| google | 訂閱, 月費, youtube, premium |
| kkbox | 訂閱, 月費 |
| 蝦皮 / shopee | vip, 訂閱 |
| LINE | 訂閱, 月費, premium |

**排除**：Apple Store / momo 購物的硬體購買

### 3.3 重複消費邏輯

- 門檻：≥5 次（半年 5 次以上 = 真正的習慣）
- **速食/壽司店按發票（visit）計**，不拆個別品項
  - 麥當勞的「餐-十塊雞 + 配-大薯 + 可樂」= 一次消費
  - 藏壽司的「30元盤 × 8」= 一次消費
  - 適用通路：麥當勞、肯德基、摩斯漢堡、漢堡王、Subway、爭鮮、藏壽司、壽司郎

### 3.4 追問設計（2 層）

**追問 A**（≥2 個訂閱才出現）：
「我的 X 個訂閱，一年花多少？哪個該留？」
→ 每個訂閱的年費 + 自問「今天才看到還會訂嗎？」

**追問 B**：
「這些重複消費可以怎麼買更聰明？」
→ 不是列品項（跟 Hook 重複），而是**具體省法 + 金額**

## 4. Hook 2：隱藏消費規律

### 4.1 分析維度

| 維度 | 計算方式 | 範例 |
|------|---------|------|
| 星期消費差異 | 每日消費按星期分組 → max/min 倍率 | 週六是週五的 7 倍 |
| 品項固定組合 | 發票內品項兩兩配對 → 出現≥3次 | 可樂+大薯 11 次 |
| 消費時段 | 按 5 個時段累計金額 | 晚上花最多 |

### 4.2 追問設計（2 層）

**追問 A**：「為什麼{最貴日}花特別多？」
→ 按品類拆解該天消費

**追問 B**：「為什麼 A+B 總是一起出現？」
→ 組合分析 + 自動化消費意識

---

## 5. Hook 3：最大品類 + 省錢 + 跟人比

### 5.1 品類選擇（複合分數）

```
composite = 金額 × 頻率^0.3 × 成長加權
```

不是「花最多的」，而是「最值得關注的」——考量金額、頻率和成長趨勢。

### 5.2 品類專屬省錢建議

| 品類 | 建議 |
|------|------|
| 速食 | App 優惠券 + 套餐 vs 單點差價 |
| 咖啡 | 品牌切換（全家 vs 路易莎）+ 自備杯 |
| 零食/飲料 | 超商改超市 + 量販包囤貨 |
| 乳製品 | 超市固定採購（超商貴 15-25%） |
| 美妝 | 週年慶集中採購 + 線上 vs 門市比價 |

### 5.3 省錢換算規則

```js
fmtComparisons(amount, stats, excludeCat)
```

**排除正在省的品類**：省速食的錢不會說「= X 次麥當勞」。

### 5.4 追問設計（2 層）

**追問 A**：「具體可以怎麼省？省多少？」
→ 品類專屬策略 + 預估年省金額 + 體驗換算

**追問 B**：「跟同類型的人比，我花得算多嗎？」
→ 受眾標籤 + 群內比較 + 品類均價 vs 同族群

## 6. 受眾分群系統

### 6.1 偵測規則（12 種標籤）

| 標籤 | 偵測方式 | 門檻 | 關鍵訊號 |
|------|---------|------|---------|
| 外食族 | 通路佔比 | ≥20% | ubereats/foodpanda/麥當勞/肯德基 |
| 超商族 | 通路佔比 | ≥25% | 7-11/全家/萊爾富 |
| 超市採購族 | 通路佔比 | ≥15% | 全聯/家樂福/美廉社 |
| 咖啡族 | 品項佔比 | ≥5% | 咖啡/美式/拿鐵/摩卡 |
| 健身/健康族 | 品項次數 | ≥15 | 雞胸/優格/豆漿/LP33/益生菌 |
| 飲料族 | 品項佔比 | ≥8% | 氣泡水/可樂/奶茶/紅茶 |
| 零食控 | 品項佔比 | ≥4% | 洋芋片/餅乾/巧克力/糖果 |
| **新手爸媽** | 品項次數 | ≥3 | **尿布/奶粉/嬰兒用品/奶瓶** |
| 美妝保養族 | 品項+通路 | ≥5 | 面膜/防曬 + 寶雅/屈臣氏 |
| 網購族 | 通路佔比 | ≥10% | momo/蝦皮/酷澎 |
| 開車族 | 品項+通路 | ≥5 | 無鉛汽油 + 中油/加油站 |
| **毛小孩家長** | 品項次數 | ≥3 | 飼料/貓砂/寵物/倉鼠 |

### 6.2 分群邏輯

- 每人可有**多個標籤**（如「外食族 + 超市採購族 + 開車族」）
- 按分數排序，**主標籤**用於群內比較
- 副標籤一起顯示，讓用戶了解自己的消費畫像

### 6.3 群內比較

同主標籤的用戶之間比較：
- 月均消費
- 品類均價（咖啡、飲料、零食、鮮食）

## 7. 排除邏輯

| 排除對象 | 原因 |
|---------|------|
| 外送平台（ubereats/foodpanda） | 發票是服務費 |
| 網購/量販（momo/蝦皮/酷澎/好市多） | 箱購單價不可比 |
| 帳單（電費/水費/電信費） | 固定支出 |
| 速食/壽司品項拆解 | 一餐的組成，用 visit 計 |
| 省錢換算用同品類 | 省麥當勞不能說「= X 次麥當勞」 |
| 「熱狗」觸發「毛小孩家長」 | 食物中的「狗」字不算 |

## 8. 語氣原則

| ❌ 不要 | ✅ 要 |
|--------|------|
| 「浪費」 | 「值得重新想想」 |
| 「壽司盤 31 次」 | 「藏壽司 8 次 $4,800」 |
| 「餐-十塊雞 9 次」 | 「麥當勞 X 次」 |
| 追問重複 Hook 內容 | 追問帶來新 insight（HOW/WHO） |
| 1 個訂閱還追問分析 | 只有 ≥2 個訂閱才追問 |
