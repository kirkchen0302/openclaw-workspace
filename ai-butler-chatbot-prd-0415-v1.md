## 3. Hook 1：訂閱 + 重複消費 + 包裝價差

### 3.1 延續 0414_v2

- 訂閱偵測（Method 1 關鍵字 + Method 1b Regex + Method 2 通路×品項配對）
- 重複消費（≥5 次門檻，速食/壽司按 visit 計）
- 追問 A：訂閱年費分析（≥2 個才出現）
- 追問 B：重複消費聰明買法

### 3.2 新增：包裝價差偵測

**偵測邏輯**：

```
同通路 + 同商品（正規化去除數量後綴）→ 不同價格 ≥ 30% 差距 → 年省金額
```

**正規化**：去除品名末尾的 `\d+(罐|入|組|盒|包|袋|瓶|片|個)`
**門檻**：價差 ≥ 30%，單價 ≥ $100，年省 ≥ $500

**範例**（Emma — 0909162902）：

| 包裝 | 單價 | 每罐均價 | vs 大包裝溢價 |
|------|------|---------|------------|
| illuma 啟賦 1 罐 | $4,034 | $4,034 | **+147%** |
| illuma 啟賦 3 罐 | $4,942 | $1,647 | +1% |
| illuma 啟賦 6 罐 | $9,780 | $1,630 | 基準 |

年省 ~$30,646（只需改下單規格）。

**追問**（當偵測到包裝價差時新增）：
「同一個商品改買大包裝能省多少？」
→ 逐一列出有價差的商品、大小包裝價格、年省金額

## 4. Hook 2：隱藏消費規律（大幅強化）

### 4.1 分析維度（0414_v2 → 0415_v1）

| 維度 | 計算方式 | 範例 | 版本 |
|------|---------|------|------|
| 星期消費差異 | 每日消費按星期分組 → max/min 倍率 | 週六是週五的 7 倍 | v2 |
| 品項固定組合 | 發票內品項兩兩配對 → 出現≥3次 | 可樂+大薯 11 次 | v2 |
| 消費時段 | 按 5 個時段累計金額 | 晚上花最多 | v2 |
| **深夜衝動消費** | **22:00-06:00 佔比 + 金額** | **22% 在深夜，$69,148** | **v1 NEW** |
| **週末溢價** | **平日 vs 週末每筆均消費** | **週末貴 32%** | **v1 NEW** |
| **頻率暴增** | **通路月頻率前後比 ≥2倍** | **酷澎 1-4→11-13 次/月** | **v1 NEW** |

### 4.2 深夜衝動消費偵測

```
lateNightTotal = SUM(amount WHERE hour >= 22 OR hour < 6)
lateNightPct = lateNightTotal / totalTimedAmount × 100
lateNightSaveable = lateNightTotal × 30%  // 30% reduction potential
```

**觸發**：佔比 ≥ 10% 時在 ranks 中顯示，佔比 ≥ 15% 時取代星期差異成為 big 數字

**追問**（佔比 ≥ 10% 時）：
「深夜消費佔 X%，可以怎麼改善？」
→ 按通路拆解深夜消費 + 具體建議（冷靜期、關推播、固定下單日）

### 4.3 週末溢價偵測

```
weekdayAvgInv = weekdayTotal / weekdayCount
weekendAvgInv = weekendTotal / weekendCount
weekendPremiumPct = (weekendAvgInv - weekdayAvgInv) / weekdayAvgInv × 100
```

**觸發**：溢價 ≥ 20% 時在 ranks 中顯示

### 4.4 頻率暴增偵測

```
befMonthly = brandFirst[store].visits / firstHalfMonths
aftMonthly = brandSecond[store].visits / secondHalfMonths
surgeRatio = aftMonthly / befMonthly
```

**觸發**：surgeRatio ≥ 2 且近期 visits ≥ 5

**追問**（有暴增時）：
「{通路}的消費為什麼突然暴增？」
→ 頻率變化 + 年化金額 + 常買品項 + 控頻建議

### 4.5 追問設計（動態優先級）

| 優先級 | 條件 | 追問 |
|--------|------|------|
| 1 | 深夜佔比 ≥ 10% | 深夜消費改善建議 |
| 1 fallback | 深夜 < 10% | 為什麼{最貴日}花特別多 |
| 2 | 有頻率暴增通路 | 暴增原因分析 |
| 2 fallback | 有固定組合 | 為什麼 A+B 總是一起出現 |

## 7. BQ 預計算架構（from insight_cards_spec）

### 7.1 三層架構

```
┌────────────────────────────────────────────┐
│ Layer 1: BQ Scheduled Query（每日 09:30）    │
│ 全量用戶預計算 → user_spending_insights 表    │
│ 增量：只算過去 7 天有新發票的用戶              │
└──────────────┬─────────────────────────────┘
               ▼
┌────────────────────────────────────────────┐
│ Layer 2: API                                │
│ GET /insights/{member_hk}                   │
│ 從快取表讀取，毫秒級回應                      │
└──────────────┬─────────────────────────────┘
               ▼
┌────────────────────────────────────────────┐
│ Layer 3: App UI                             │
│ 3 張卡片 + Chatbot，模板填入數字               │
│ 金額全用年度累計，最大化衝擊力                  │
└────────────────────────────────────────────┘
```

### 7.2 快取表 Schema

```sql
CREATE TABLE IF NOT EXISTS `user_spending_insights` (
  member_hk BYTES,
  computed_at TIMESTAMP,
  total_12m INT64,
  invoice_count INT64,
  -- Card 1: Top store
  c1_store STRING,
  c1_amount INT64,
  c1_pct FLOAT64,
  -- Card 2: Savings
  c2_total_saving INT64,
  c2_saving_freq INT64,
  c2_saving_sub INT64,
  c2_saving_late_night INT64,
  -- Card 3: Timing
  c3_peak_hour INT64,
  c3_peak_hour_amount INT64,
  c3_peak_month STRING,
  c3_peak_month_amount INT64,
  c3_weekend_premium_pct FLOAT64
);
```

### 7.3 成本估算

| 項目 | 估算 |
|------|------|
| 全量首次計算（270 萬用戶） | ~$50-100（一次性） |
| 每日增量更新（20-50 萬有新發票） | ~$5-15/天 |
| 建議 | 先跑 1 萬用戶驗證，再決定全量 |

## 8. 3 張洞察卡片（from insight_cards_spec）

### 卡片 1：「你最大的隱藏花費是什麼？」

找用戶過去 12 個月消費金額最高的店家，年度累計呈現。

### 卡片 2：「你一年可以省多少？」

加總三種節省機會：
- 高頻減頻（同店 ≥12 次且單次 <$500 → ×30%）
- 訂閱清理（同金額出現 ≥3 月 → 年化全額）
- 深夜衝動（22:00-06:00 → ×30%）

### 卡片 3：「你的錢都在什麼時候不見的？」

找最大消費時段 + 最貴月份 + 週末溢價，擇最驚訝的顯示：
1. 深夜佔比 ≥ 10% → 時段版
2. 週末溢價 ≥ 20% → 週末版
3. 都不符合 → 月份版

## 10. 排除邏輯（延續 0414_v2 + 新增）

| 排除對象 | 原因 |
|---------|------|
| 外送平台（ubereats/foodpanda） | 發票是服務費 |
| 網購/量販（momo/蝦皮/酷澎/好市多） | 箱購單價不可比 |
| 帳單（電費/水費/電信費） | 固定支出 |
| 速食/壽司品項拆解 | 用 visit 計 |
| 省錢換算同品類 | 省麥當勞不能說「= X 次麥當勞」 |
| **包裝價差小於 30%** | **不顯著** |
| **包裝單價 < $100** | **低價商品不值得比較** |
