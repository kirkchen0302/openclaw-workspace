## 1. 版本差異摘要（vs 0411_v3）

| 面向 | 0411_v3 | 0414_v1 |
|------|---------|---------|
| Hook 數量 | 4-5 個 | **6 個**（+ subscription 可達 7） |
| Hook 選擇 | 動態評分競爭 | **標準 6 個 + 評分填補** |
| 品類選擇 | 飲料優先 / 全品類 | **複合分數**（金額 × 頻率 × 成長率） |
| 跟別人比 | 同類型通路內 | **FMCG 跨超商+超市，店特有品只在同類型** |
| 隱藏規律 | 無 | **新增：星期消費差異 + 品項組合偵測** |
| 開場訊息 | 3 句漸進 + teaser | **1 句簡潔引導** |
| Hook 按鈕 | 有時引用未知金額 | **每個按鈕一眼就能懂** |
| 追問方向 | 部分列品項 | **全部分析導向**（趨勢/佔比/比較/優化） |

## 2. Hook 架構

### 2.1 開場

```
「嗨，我已經看完你的 {N} 張發票和所有品項明細了。選一個你好奇的問題，我來幫你分析 👇」
```

一句話 + 直接顯示所有 Hook 按鈕。不做 teaser、不做漸進出場。

### 2.2 標準 Hook 1-6（有品項資料時）

| # | Hook 按鈕 | 類型 ID | 觸發 | 分析維度 |
|---|---------|---------|------|---------|
| 1 | 「我有哪些東西一直在重複買？」 | `autopay` | 品項≥3次且≥3個 | 重複品項 × 通路 × 累計金額 |
| 2 | 「我的『{品類}』花了多少？」 | `items` | 有品項資料 | 品類複合分數 × 趨勢 × 佔比 |
| 3 | 「同樣的東西，別人怎麼買？」 | `benchmark` | 有超商/超市消費 | FMCG 跨通路 + 店特有同類型 |
| 4 | 「我的消費有什麼隱藏規律？」 | `pattern` | 發票≥30張 | 星期差異 + 品項組合 + 時段 |
| 5 | 「怎麼省最有感？」 | `save` | 有≥2個策略 | 排序過的省錢方案 |
| 6 | 「我的訂閱還值得嗎？」 | `subscription` | ≥2個訂閱 | 月費 → 年度加總 |

### 2.3 Hook 2 品類選擇（複合分數）

不是純看金額最高，而是用複合分數：

```
composite = 金額 × 頻率^0.3 × 成長加權
```

成長加權：
- 頻率成長 >30%：×1.5
- 頻率成長 >100%：×2.0

**效果**：找出「你不知道但正在發生」的品類，不只是「花最多的」。

| 用戶 | 純金額 Top | 複合 Top | 為什麼複合更好 |
|------|----------|---------|-------------|
| 0972 | 美妝保養 | **乳製品** | 81 次 +112% 你沒注意到的膨脹 |
| 0988 | 生鮮肉品 | **乳製品** | 肉品在下降，乳製品 +38% |
| 0983 | 咖啡 | **乳製品** | 咖啡 -22%，乳製品 +32% |

## 3. 各 Hook 詳細規格

### 3.1 Hook 1：「我有哪些東西一直在重複買？」（AUTOPAY）

**大數字**：重複品項數量（如「51 個」）
**排行**：品項名 + 次數 + 累計金額 + 通路（不用月費框架）

**追問 A**：哪些想保留、哪些該檢視？
- 深追問：減掉順手買的能省多少？
- 深追問：每家店的固定菜單是什麼？

**追問 B**：哪些品項是我沒意識到一直在買的？
- 深追問：這些加起來一年多少？
- 深追問：有更划算的買法？

### 3.2 Hook 2：「我的『{品類}』花了多少？」（ITEMS）

**品類動態選擇**：用複合分數挑最值得關注的品類
**大數字**：該品類累計金額
**排行**：品類排行（含成長率 📈/📉 標記）

**追問 A**：這個花費有在增加嗎？一年要花多少？
- 深追問：這樣的花費合理嗎？（佔總消費比例 + 日均）
- 深追問：跟其他品類比起來呢？（品類排行）

**追問 B**：跟別人比，我的「{品類}」算多嗎？
- 深追問：有什麼方法可以在這個品類省錢？
- 深追問：哪個品類成長最快？

### 3.3 Hook 3：「同樣的東西，別人怎麼買？」（BENCHMARK）

**比較邏輯**：

| 品類 | 比較範圍 | 原因 |
|------|---------|------|
| 瓶裝飲料 | 超商 + 超市 合併 | FMCG，同產品跨通路 |
| 乳製品 | 超商 + 超市 合併 | FMCG |
| 零食 | 超商 + 超市 合併 | FMCG |
| 咖啡 | 只在超商內 | 各店自有品牌 |
| 鮮食 | 只在超商內 | 各店自有品牌 |

**群體基準值**（從 9 位測試者計算）：

| Key | 群體均價 | 最低 | 最高 |
|-----|---------|------|------|
| 瓶裝飲料@零售 | $29 | $22 | $39 |
| 乳製品@零售 | $52 | $32 | $80 |
| 零食@零售 | $46 | $30 | $53 |
| 咖啡@超商 | $55 | $35 | $68 |
| 鮮食@超商 | $48 | $36 | $62 |

**追問 A**：我在哪裡花得比別人多？
- 深追問：怎麼做到跟別人一樣省？
- 深追問：差距最大的是什麼品項？

**追問 B**：我有比別人更會買的地方嗎？
- 深追問：我的省錢方法可以套用到哪？
- 深追問：整體來看我算會買嗎？

### 3.4 Hook 4：「我的消費有什麼隱藏規律？」（PATTERN）

**計算邏輯**：

**1. 星期消費差異**
```
每天消費 → 按星期分組 → 計算日均
找出最高和最低的星期 → 計算倍率
```
- 觸發條件：max/min 倍率 ≥ 2

**2. 品項組合偵測**
```
每張發票的品項 → 兩兩配對 → 計算共同出現次數
找出出現 ≥ 3 次的固定組合
```
- 排除外送服務費和「其他」分類

**3. 消費時段**
```
每張發票的時間 → 分到 5 個時段 → 累計金額
找出花最多的時段
```

**追問 A**：為什麼{最貴日}花特別多？
- 深追問：降到平均能省多少？（年化金額 + 體驗換算）
- 深追問：有衝動消費日嗎？（超日均 2 倍的日子）

**追問 B**：「A」和「B」為什麼總是一起出現？（最常見品項組合）
- 深追問：這些組合一年花多少？
- 深追問：打破組合有什麼好處？

### 3.5 Hook 5：「怎麼省最有感？」（SAVE）

5 種策略按「省最多 × 最不費力」排序：

| 策略 | 觸發條件 | 難度 |
|------|---------|------|
| 超商改超市 | 超商≥10次 + 有超市 | 低 |
| 控制均價爬升 | 有均價上升品牌 | 低 |
| 減少最高頻通路 | 每週≥2次 | 中 |
| 合併同類通路 | 同類≥2個品牌 | 低 |
| 控制新習慣 | 成長>200% | 中 |

**追問 A**：最無痛的第一步是什麼？
- 深追問：做了之後下一步呢？
- 深追問：會不會影響生活品質？

**追問 B**：全部做到的話呢？
- 深追問：哪個 CP 值最高？
- 深追問：我的消費整體算健康嗎？

### 3.6 Hook 6：「我的訂閱還值得嗎？」（SUBSCRIPTION）

觸發：≥2 個訂閱
偵測：嚴格 keyword match（訂閱/月費/會員訂閱/Uber One/pandapro）

**追問 A**：這些訂閱都還值得嗎？
- 深追問：如果全部取消能省多少？
- 深追問：哪個訂閱最值得重新想想？

**追問 B**：放大到一年來看呢？
- 深追問：哪些訂閱我可能用不夠？
- 深追問：如果暫停一半，能省多少？

## 4. 排除邏輯

| 排除對象 | 原因 | 影響範圍 |
|---------|------|---------|
| 外送平台（ubereats/foodpanda） | 發票是服務費，非餐費 | 所有品項分析、opener、排行 |
| 網購/量販（momo/蝦皮/酷澎/好市多） | 箱購單價不可比 | BENCHMARK 比較 |
| 帳單（電費/水費/電信費） | 固定支出，非行為問題 | 隱形消費分析 |
| 7-11 咖啡 vs 星巴克咖啡 | 不同定位，不能比價 | 只在同類型通路內比 |
| 壽司盤/配餐薯條 | 一餐的組成部分 | 不獨立算月費 |

---

## 5. 省錢比較框架（消費心理學）

| 層級 | 心理機制 | 範例 |
|------|---------|------|
| Layer 1: 自我錨點 | Self-anchoring | 「等於你去 7-11 70 次」 |
| Layer 2: 體驗替代 | 體驗 > 物質 | 「一趟東京 3 天」「聚餐 20 次」 |
| Layer 3: 日均損失 | Loss Aversion | 「每天 $28 = 一杯路易莎拿鐵」 |

## 6. 語氣原則

| ❌ 不要 | ✅ 要 |
|--------|------|
| 「浪費」「過高」 | 「值得重新想想」「算多嗎？」 |
| 「$51/月在爭鮮」（壽司盤算月費） | 「9 次 共 $360 在爭鮮」 |
| 「你的外送費高達」 | 「你跟 XX 的關係很穩定」 |
| 「建議你減少」 | 「你自己決定」 |
| Hook 引用用戶不知道的金額 | 每個按鈕一眼就能懂 |
